# Postman API Testing Project

![Postman Tests Screenshot](screenshot/image.png)

Этот проект создан для демонстрации автоматизированного тестирования API с помощью Postman. Он включает в себя простое приложение на Python (FastAPI), которое предоставляет CRUD операции для управления каталогом книг. В проекте также содержится коллекция и окружение Postman для тестирования.

## Структура проекта

* `app.py` - Основной файл приложения на FastAPI, реализующий REST API.
* `requirements.txt` - Зависимости проекта (перечислены необходимые Python-библиотеки).
* `Book_API_Collection.json` - Коллекция Postman с 5-ю запросами и автотестами.
* `Book_API_Environment.json` - Окружение Postman (содержит переменную `userId`).

## Что реализовано

* **API Endpoints:**
  * `POST /books` - Создание книги.
  * `GET /books` - Получение списка всех книг.
  * `GET /books/{book_id}` - Получение информации о конкретной книге.
  * `PUT /books/{book_id}` - Обновление информации о книге.
  * `DELETE /books/{book_id}` - Удаление книги.

* **Тестирование в Postman:**
  * Использование коллекции с переиспользуемыми переменными (`baseUrl`, `bookId`, и др.).
  * **Pre-request Script:** В запросе `POST /books` автоматически генерируется случайное название книги.
  * Настроен **Environment** с переменной `userId`, которая передается в заголовке `X-User-Id` (эмуляция авторизации/идентификации).
  * В каждом запросе присутствует минимум 2 проверки (ответа сервера `< 500ms`, ожидаемые HTTP статусы `200\201`, соответствие полей JSON-структуре).
  * Корректная работа через **Postman Collection Runner** (тесты проходят друг за другом без вмешательства пользователя).

## Инструкция по локальному запуску сервера

1. Убедитесь, что у вас установлен Python 3.
2. Склонируйте репозиторий и перейдите в папку с проектом.
3. Создайте и активируйте виртуальное окружение:
   ```bash
   python3 -m venv venv
   source venv/bin/activate  # Для Linux/macOS
   # venv\Scripts\activate   # Для Windows
   ```
4. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
5. Запустите API сервер:
   ```bash
   python app.py
   ```
   *Сервер будет доступен по адресу `http://127.0.0.1:8000` (также документация Swagger доступна по `/docs`).*

## Запуск тестов Postman

1. Откройте приложение Postman (рекомендуется Desktop версия или Desktop Agent для доступа к `127.0.0.1`).
2. Нажмите кнопку **Import** и загрузите два файла из корня этого проекта: `Book_API_Collection.json` и `Book_API_Environment.json`.
3. В правом верхнем углу интерфейса Postman выберите загруженное окружение **BookAPI Env**.
4. В левом меню нажмите на коллекцию `Book API CRUD Testing` -> `Run collection`.
5. Запустите тесты (нажмите `Run Book API CRUD Testing`) и убедитесь, что все тесты отображаются как Passed (зеленые галочки).
